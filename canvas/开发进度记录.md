### 功能二：紧急程度自定义 — 阶段A 数据层(TDD) ✅

**需求概述**：将任务紧急程度的文本标签和颜色从硬编码改为用户可配置，支持4个级别（紧急/较紧急/一般/不紧急）的自定义。

**设计参考色值**：
| 级别 | 默认标签 | 默认描述 | 默认颜色 |
|---|---|---|---|
| 0 | 紧急 | 需要立即处理 | #FF0000 (红) |
| 1 | 较紧急 | 近期需要完成 | #FF7A00 (橙) |
| 2 | 一般 | 正常推进即可 | #FFD700 (黄) |
| 3 | 不紧急 | 可以稍后处理 | #007BFF (蓝) |

**阶段A 已完成 — 数据模型 + 单元测试**：

| 文件 | 变更类型 | 说明 |
|---|---|---|
| `task-model/.../model/UrgencyLevelConfig.kt` | 新增 | UrgencyLevelConfig + UrgencyConfig 数据类，纯 Kotlin 正则 JSON 解析 |
| `task-model/.../util/ColorUtils.kt` | 新增 | HEX→ARGB 颜色解析工具（纯 Kotlin，无 Android 依赖） |
| `task-model/.../model/UrgencyConfigTest.kt` | 新增 | 13 个单元测试 |
| `task-model/.../util/ColorUtilsTest.kt` | 新增 | 3 个单元测试 |

**关键技术决策**：
- task-model 模块无 org.json 依赖 → 自实现纯 Kotlin 正则 JSON 解析器
- task-model 模块无 android.graphics.Color → 自实现 ColorUtils.parseHexToArgb()
- 每个级别 = 一个颜色 + 一个文本，不做同级别的色彩渐变序列

## 四、架构备忘

```
项目结构：Kotlin + Jetpack Compose Android
├── app/ (Android UI 层，可依赖 Android Framework)
│   ├── MainScreen.kt (NavHost + BottomNavigation 5 tab)
│   ├── ui/quote/QuoteSettingScreen.kt
│   ├── ui/mine/settings/SettingsScreen.kt
│   ├── core/ui/StyleConstants.kt (AppColors, AppSpacing, AppShapes)
│   └── widget/ (WidgetTaskStyle, WidgetManager — Android View 系统)
└── task-model/ (纯 Kotlin 模块，无 Android Context)
    ├── model/ (UrgencyLevelConfig, Task, Note 等)
    ├── repository/ (TaskRepository — DataStore 持久化)
    ├── viewmodel/ (TaskViewModel)
    └── util/ (TaskUtils, ColorUtils, NoteFilterUtils)
```

**数据流**：DataStore → Repository Flow → ViewModel State → Compose UI / Widget
**紧急程度配置流**：DataStore JSON → UrgencyConfig → CompositionLocal (Compose) / @Volatile (Widget)
