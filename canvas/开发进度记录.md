### 功能四：紧急程度颜色实时响应修复 ✅

**问题**：用户在设置页修改紧急程度颜色后，已创建的任务颜色不会实时更新

**根因**：`TaskUtils.getUrgencyColor()` 读取静态变量，但设置页修改后未同步到 `TaskUtils`

**修复方案**：
- `MainScreen` 添加 `LaunchedEffect`，当 `urgencyConfig` 变化时同步到 `TaskUtils`
- 将 8 处 Compose 组件中的 `TaskUtils.getUrgencyColor()` 替换为 `LocalUrgencyConfig` 响应式读取
- 清理 `TaskUtils` 中未使用的 `urgencyColorMap` 死代码

**涉及文件**：
- `app/.../MainScreen.kt`
- `app/.../ui/TaskItemCard.kt`
- `app/.../ui/CreateScreen.kt`
- `app/.../ui/components/CreateTaskBottomSheet.kt`
- `app/.../ui/view/month/MonthCalendar.kt`
- `app/.../ui/widget/WidgetMainScreen.kt`
- `task-model/.../util/TaskUtils.kt`

### 功能五：紧急程度设置国际化修复 ✅

**问题**：设置页面中 `urgency_settings_title` 等文字显示为资源键名，未正确进行中英文切换

**根因**：`LocalizedStrings.get()` 会自动添加 `i18n_` 前缀查找资源，但 urgency 相关 key 在 `strings.xml` 中没有 `i18n_` 前缀

**修复方案**：在 `values/strings.xml` 和 `values-en/strings.xml` 中添加 `i18n_` 前缀的 urgency 相关翻译

**涉及 key**：
- `i18n_urgency_settings_title`
- `i18n_urgency_levels`
- `i18n_edit_urgency_level`
- `i18n_label`、`i18n_description`、`i18n_color_hex`
- `i18n_reset_to_default`、`i18n_save`、`i18n_cancel`

## 四、架构备忘

```
项目结构：Kotlin + Jetpack Compose Android
├── app/ (Android UI 层)
│   ├── MainScreen.kt (NavHost + CompositionLocalProvider)
│   ├── ui/mine/settings/UrgencySettingsScreen.kt (设置页面)
│   ├── ui/view/today/UrgencyCircle.kt (动态颜色)
│   ├── core/ui/StyleConstants.kt (LocalUrgencyConfig)
│   └── widget/ (WidgetTaskStyle, WidgetManager)
└── task-model/ (纯 Kotlin 模块)
    ├── model/UrgencyLevelConfig.kt (数据模型)
    ├── util/ColorUtils.kt (颜色工具)
    └── util/TaskUtils.kt (动态配置注入)
```

**数据流**：
```
DataStore JSON → UrgencyConfig → 
  ├─→ CompositionLocalProvider (Compose UI)
  └─→ WidgetTaskStyle.updateFromConfig() (Widget)
```
