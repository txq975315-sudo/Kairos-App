## 一、一句话产品定位（定稿口径）

**Kairos = 每日计划秩序（待办）+ 每日灵感记录（随笔）+ 周/月桌面可视化（小组件）**  
**双轨并存：理性计划 + 感性记录，极简闭环；纯本地、无账号、无云同步。**

（与 README「质量时间」叙事可同时使用：前者偏**功能边界**，后者偏**品牌故事**。）

## 二、求职清单 · 面试前必对齐项（勾选）

### 产品边界（决策记录1 · 做 / 不做）

- [ ] 能口述「**坚决不做**」清单：云同步 / 登录 / 社交协作 / 复杂标签体系 / 小组件「每日一句来源」独立配置入口（定稿已删）等
- [ ] 能说明 **V1 View 只做数据呈现**，不做「模式洞察规则引擎」（延后 V2）
- [ ] 能说明 **删除统一口径**：左滑 + Snackbar 约 3 秒撤销，超时物理删除；定稿**无回收站**（与部分历史实现可能不一致，见下文「诚实口径」）
- [ ] 能说明 **每日一句**：定稿为 **Today 页顶部唯一入口**（避免多入口混乱）
- [ ] 能说明 **小组件**：定稿为 **只读展示 + 分区域跳转 App**；布局 / 条数 / 刷新 / 背景可配
- [ ] 能说明 **今日待办硬规则**：单日 **最多 12 条**，超出弹窗拦截（极简克制）
- [ ] 能说明 **晨间复盘**（若已实现）：昨日数据锁定、复盘操作「带入今日 / 标记逾期」等（以定稿文档为准）

### Essay 与数据（决策记录2 + 3）

- [ ] **默认随手记 / 随手记**：FAB 进编辑页默认「随手记」，不强制课题弹窗；零摩擦速记
- [ ] **Inbox**：随手记保存后进 Inbox（约 **15 天**缓冲），到期转永久杂记；深度课题可走直接发布路径（与定稿状态机对齐表述）
- [ ] **关联课题**：主分类单选 + `linked_categories` 多选（不含 freestyle、不与主分类重复）；过滤规则为「主或关联命中即展示」
- [ ] **原地展开**：View / 视察场景点击卡片原地展开，正文浏览态不可直接编辑（防误触）
- [ ] **课题管理**：Mine 内 **六宫格** 汇总 → 主课题编辑；**自由随想锁定**，其余 5 个主课题可改名 / Emoji / 引导句；二级最多 8 个

### View（决策记录2 + 3）

- [ ] **三时间镜头**：今日 / 本周 / 本月；同一页内切换，减少跳转
- [ ] **本周左右页**：Todo 与 Essay **日期严格对齐**、同步滚动（共用一个滚动状态的设计决策）
- [ ] **今日分区**：上半 Todo（条数与排序以定稿为准）、下半最新笔记等（决策记录2 有具体数字，面试可背）

### Mine · 全部记录（决策记录3）

- [ ] **双 Tab**：「打卡日历」与「数据统计」分离（感性痕迹 vs 理性指标）
- [ ] **打卡图标体系**：完美 / 优秀 / 加油 / 新芽 / 文档记录者 / 空白 —— 条件为「任务完成度 + 是否有随笔」组合；**无惩罚性图标**
- [ ] **统计自定义**：**13 选 8** 指标、拖拽排序、卡片轮播与长按管理

### 视觉与国际化（决策记录2 + 3）

- [ ] **素净融入**：无独立卡片大阴影、分隔靠细线与留白；例外：Mine **心情卡 / 个人卡** 极轻阴影
- [ ] **去灰色化 / 暖棕系**：页面底色、分隔线、次要文字等（决策记录2 色彩表）
- [ ] **国际化五步**：扫描 → 清单 → strings → 切换逻辑 → **Lint / CI 防退化**（决策记录3）

### 技术表述（决策记录1 与仓库实际）

- [ ] 定稿文档写过 **Jetpack Glance**；若代码路径为 **RemoteViews**，面试话术：**「小组件层定稿考虑过 Glance，落地选用 RemoteViews 服务当前布局与兼容性」**（按你仓库真实情况二选一，勿编造）
- [ ] 定稿曾写 **禁止网络 / 通知等权限**；若 `AndroidManifest` 仍含通知等权限：**「个人迭代中曾接入提醒能力，与最新定稿的极简边界存在张力，面试中如实说明当前包体状态」**

## 三、与代码 / 历史版本可能不一致时的「诚实口径」

面试官若对照你手机或 GitHub，可能看到与《决策记录1》不完全一致的功能（例如历史 commit 中的回收站、小组件上可勾选任务等）。建议统一用下面框架回答：

> 「产品侧有一份**书面决策记录**作为需求真源，迭代过程中部分能力先上线验证，**最新定稿**里对 {回收站 / 小组件只读 / 每日一句入口 / 重复任务} 做了收敛或调整。  
> 我求职时以**定稿文档**讲产品判断；若您看到我仓库里还有旧实现，那是**尚未合并回定稿**或**刻意保留的技术实验**，我可以逐项说明差异和取舍原因。」

这样既不甩锅，也不被质疑「简历造假」。

## 四、简历 / 作品集可引用的「硬决策」金句（直接摘用）

1. **「把开放性问题变成选择题」** — 随手记默认 + Inbox 缓冲，深度记录再选课题。  
2. **「视察不打扰创作」** — View 里原地展开，减少跳 Essay 编辑页。  
3. **「左右脑要在同一天对齐」** — 本周视图 Todo 与 Essay 同步滚动。  
4. **「桌面是最高频屏幕」** — 4 尺寸 × 多布局 + 可读性优先的背景策略。  
5. **「数据驱动之前先定义不做」** — 定稿明确不做规则引擎、不做云同步，先把「看」和「记」做透。  
6. **「国际化不是翻译，是工程流程」** — 扫描、资源、切换、Lint 防退化五步。  
7. **「同一天要在同一根横线上」** — 本周 View 左 Todo、右 Essay，**同步滚动**才谈得上对照（见 `01` 故事 D）。  
8. **「数据可以是镜子，不该是鞭子」** — 打卡双 Tab、图标阶梯**无惩罚性 ❌**、统计 **13 选 8**（见 `01` 故事 E）。
