# 自荐信、跟进邮件、社交媒体话术模板

> 这份是各种"对外发送"的文本模板，按场景排版。
> 用法：复制 → 替换 {占位符} → 发送。

---

## 一、自荐信（Cover Letter）— 邮件投递专用

### 通用版（推荐）

**主题**：应聘 {公司名} {岗位名} — 应届产品候选人 {你的姓名}（GitHub 附独立 App 项目）

```
{招聘官称呼} 您好，

我是 {学校}{专业} {年级} 在校生 / 应届毕业生 {姓名}，
今天通过 {渠道：贵司官网 / Boss / 学长内推} 看到了 {岗位名} 的招聘需求，
非常希望成为团队中的产品助理。

不同于大多数同期应届生，我用过去 2 个月时间从 0 独立设计并开发了
一款 Android 端时间管理 App「Kairos」（GitHub: {链接}）：

  · 5 个一级模块、30+ 页面、4 种桌面小组件、34,000+ 行代码
  · 6 次完整版本迭代，覆盖需求洞察 → 信息架构 → 视觉规范 → 工程交付 → 复盘全流程
  · 主导信息架构重构（把"日记"拆解为 6 个心理学主题），把"每天写什么"
    从开放问题变成选择题，显著降低用户启动门槛

之所以投递贵司，是因为我注意到 {一句具体的、贵司近期产品 / 业务亮点}，
我相信这份{侧重用户场景 / 数据驱动 / xx 方向}的产品风格，
和我在 Kairos 里践行的"用具体场景检验功能"的方法论高度契合。

附件是我的简历和作品集 PDF，期待有机会和您聊聊。

  联系电话：{手机号}
  邮箱：{邮箱}
  作品集：{Notion 链接}
  GitHub：{仓库链接}

祝好，
{姓名}
{日期}
```

> 关键写作原则：
> 1. 主题行带"GitHub 附独立 App 项目"——比同类邮件更容易被点开
> 2. 第二段必须有 3 行 bullet 数据，HR 5 秒能扫到亮点
> 3. 第三段必须有"针对这家公司的一句话"，证明你不是群发
> 4. 末尾联系方式齐全，方便 HR 直接拨电话

## 二、Boss 直聘 / 拉勾 等平台首条招呼语

> 这些平台的 HR 每天收 100+ 条"您好"，**第一条消息必须 1 秒抓住眼球**。

### 通用模板（适合大多数岗位）

```
您好！应届产品候选人 {姓名}，独立交付过 1 款 Android App
（5 模块 / 30+ 页面 / 4 种小组件 / 34000+ 行代码 / 6 次迭代）。
对贵司 {岗位名} 很感兴趣，能否方便发份完整 JD 看一下？
我可以同时把简历 + 作品集发您。
```

### 突出"对这家公司有研究"版

```
{HR 您好}！我看到贵司在做 {具体产品 / 业务方向}，
作为深度用户我有一些观察想分享 ——
我自己也独立做过一款 Android App「Kairos」（GitHub: {短链}），
和贵司在 {某个角度} 有相通的产品理念。
请问 {岗位名} 现在还在招吗？我能发份简历 / 作品集给您吗？
```

### 学长学姐内推版

```
{学长姐姓名} 您好！我是 {学校}{专业} 学弟 / 学妹 {姓名}，{年级}。
听说您在 {公司} 做 {岗位}，方便请教您几个问题吗？
也想问问您部门 {岗位名} 是否还有内推名额？
我自己独立做过一款 Android App，附作品集链接 {链接}，
方便的话我先发份简历给您看？
```

> 三个原则：
> - **不超过 3 行**
> - **永远先给"我有什么"再问"你能给我什么"**
> - **给个超链接 / 项目名称，对方能 1 秒判断你不是水的**

## 三、跟进 / Catch-up 邮件

### 投递后 7 天没回复

**主题**：Re: 应聘 {岗位名} — 简历跟进 {你的姓名}

```
{HR 称呼} 您好，

我是 {日期} 投递贵司 {岗位名} 的应届生 {姓名}。
冒昧打扰，想和您简单确认一下我的简历是否已收到。

如果方便的话，能否告知我大致的简历筛选周期？
若已经进入流程，我会耐心等待；若暂时还没看，
我可以再做一个 1 分钟自我介绍视频补充给您。

谢谢您的时间。

  电话：{手机号}    邮箱：{邮箱}
  作品集：{链接}
{姓名}
```

> 注意：
> - 跟进只发一次，**不要追第二次**
> - 主动提供新增价值（"我可以再做个视频补给您"），不要纯催

---

### 一面后 5 天没回复

**主题**：感谢 {岗位名} 一面机会 — {你的姓名}

```
{面试官称呼} 您好，

我是 {日期} 面试 {岗位名} 的 {姓名}。
本次面试您和我聊到的 {具体话题，证明你记得}，
回去之后我又做了一些延伸思考 / 阅读 / 实践，
{用一句话说明你延伸思考的成果，比如新画了一张产品流程图、
拆解了一款相关 App、读完一本相关的书}。

附上我的延伸输出：{链接 / PDF}。

不论后续结果如何，
本次交流让我对 {贵司 / 团队 / 产品方向} 有了更具体的理解，
非常感谢您和团队的时间。

期待后续。

{姓名}
{手机号} | {邮箱}
```

> **这是 90% 应届生不会做的差异化动作**，做对了能让你从同期候选人里"被记住"。

## 四、领英 / 脉脉 / 微信 拓展人脉的话术

### 联系学长学姐 / 行业前辈（信息咨询型，不直接要内推）

```
{学长姐 / 老师 / 前辈} 您好！我是 {学校}{专业} 的 {姓名}，{年级}。
我在准备求职 {方向}，
看到您在 {公司}{岗位} 工作 / 之前分享过 {某篇文章 / 某次直播}，
想请教您一个具体问题：
{放一个具体、能 30 秒回答的问题，不要问"怎么准备产品面试"这种大问题}

如果您方便的话，能不能花 3-5 分钟回我一下？
非常感谢！如果不方便也完全理解。
```

> **核心心法**：
> - 不要一上来就要内推（对方反感）
> - 先用一个**具体可回答**的问题建立联系
> - 对方回答后再考虑下一步

### 联系前辈第二步：建立长期关系

```
{前辈} 您好！再次感谢您上次的回答，对我帮助很大。
我后来按照您说的 {做了什么具体行动}，
{有什么具体收获 / 新问题}。

冒昧问一下，您部门最近还在招 {岗位名} 吗？
如果方便的话，我能把简历发您参考一下吗？
```

## 五、被拒之后的高情商回复模板

### 笔试 / 一面被刷

```
{HR 您好}：
非常感谢通知，也感谢贵司给我这次面试机会。
能够和 {面试官姓名 / 面试 + 流程} 交流是我的收获。
如果方便的话，能否请您简单告知一下被淘汰的主要原因？
我希望针对性地补强、下次再投递更好的版本。

无论是否方便回复，再次感谢！
祝贵司业务越做越好。
{姓名}
```

### 终面被刷（OFFER 没拿到）

```
{HR / 面试官} 您好，
非常感谢您和团队这一路给我的机会，
非常遗憾这次没能加入团队。
本次过程让我学到了很多，
也让我对 {贵司 / 行业} 有了更具体的理解。

如果未来贵司有 {同类岗位 / 实习机会} 重新开放，
还希望您能想到我，我会持续关注。
祝贵司业务顺利！

{姓名}
{手机号} | {邮箱}
```

> 被刷之后保持高情商不只是礼貌，**HR 圈很小**——你今天回得体，
> 半年后她跳到另一家可能就直接想到你。

## 六、Offer 谈判与决策邮件

### 收到 Offer 后请求宽限期（决定时间）

```
{HR 您好}：
非常感谢贵司给我这个 Offer，看到时真的很激动。
基于这是我职业生涯的第一份工作，我希望能慎重一些，
能否请您给我 {3-5} 个工作日时间，让我和家人 / 学校老师 / 前辈
做一些最后的沟通？

我会在 {具体日期} 之前给您明确答复。
非常感谢您的理解！

{姓名}
```

### 接受 Offer 邮件

```
{HR 您好}：
经过认真考虑，我正式接受贵司 {岗位名} 的 Offer，
非常感谢您和团队给我这个机会，期待早日加入！

我已仔细阅读 Offer 中的所有条款，确认没有疑问。
关于入职准备，请问我需要在 {入职日} 之前
准备 / 完成哪些事项（背调材料 / 体检 / 入职文件签署）？

期待您的指引。
{姓名}
```

### 婉拒 Offer 邮件（必须发，行业很小）

```
{HR 您好}：
非常抱歉这么晚回复您。
经过深思熟虑，我最终选择了一个更贴合我个人长期方向的 Offer，
非常遗憾这次没能加入贵司团队。

整个面试过程让我学习到了很多，
{面试官姓名 / 团队} 给我留下了非常好的印象。
也希望未来如果有合适的机会，能再次和贵司有合作的可能。

再次感谢您的时间和耐心，祝贵司业务越做越好！

{姓名}
```

## 七、社交媒体（小红书 / 即刻 / 微博）求职帖模板

> 0 内推、0 实习、海投效率低时，**主动在社交媒体"广撒网"是隐形高 ROI 渠道**。

### 求职贴模板

**配图**：1 张项目截图 + 1 张"我是谁"的自我介绍卡片

**文案**：

```
🎓 求职 ｜ 应届产品助理 / 初级 PM 找工作

📌 一句话定位
{学校}{专业} {年级 / 应届}，
独立从 0 交付 1 款 Android App「Kairos」
（5 模块 / 30+ 页面 / 4 种小组件 / 34000+ 行代码 / 6 次迭代）

📌 我能做什么
- 用户洞察 + 信息架构 + 原型设计
- 写 PRD、画功能流程、做竞品拆解
- 看得懂 Kotlin / SQL，能和研发同频沟通

📌 想找什么
- 城市：{城市} ／ 远程都可
- 方向：用户产品 / 工具产品 / 内容社区类优先
- 薪资：可议（应届起薪可参考市场区间）

📌 联系方式
- 简历 / 作品集：评论 / 私信发链接
- 微信：{号}（备注"求职"）

求转发！求 mention 朋友！
{tag 一些热门话题：#应届生求职  #产品经理  #互联网求职  #校招}
```

### 发布平台优先级

| 平台 | 适合度 | 注意事项 |
|------|--------|----------|
| 即刻 | ⭐⭐⭐ | 互联网圈活跃，HR / Leader 比例高 |
| 小红书 | ⭐⭐⭐ | 字节 / 美团 / 小红书内部员工常刷，话题热 |
| 脉脉 | ⭐⭐⭐ | 直接接触在职 HR |
| 微博 | ⭐⭐ | 适合带话题 #校招 #求职 |
| 知乎 | ⭐⭐ | 用回答相关求职问题"软曝光"，比直接发帖好 |

## 八、最高级别的"求职故事化"——做一篇深度文章

> 如果你想真正在求职市场被记住，**写一篇关于 Kairos 的深度复盘文章**会比任何简历都有效。

### 推荐写作结构（即可发到小红书 / 公众号 / 即刻 / 知乎）

**标题模板**（任选其一）：
- 「我用 2 周做了一款 Android App，准备拿它去面试产品经理岗」
- 「0 实习应届生求职：我把"独立做一个 App"当成了我的实习」
- 「为什么我做的这款时间管理 App，用 6 个心理学主题代替自由分类？」

**正文结构**：

```
1️⃣ 开头钩子（一段话讲清楚）
"我是个 0 实习的应届生，没法用「在 XX 公司实习过 X 月」开头我的简历。
所以我决定花 2 周时间，用一个完整的项目证明：我已经具备了产品助理需要的能力。"

2️⃣ 项目介绍（200 字概览）
- 是什么
- 为什么做
- 现在长什么样（3 张关键截图）

3️⃣ 我的产品决策故事（重头戏，2-3 个故事）
- 为什么把日记改成 6 个固定主题
- 为什么紧急程度颜色要做实时同步
- 为什么把每日一句和笔记打通

4️⃣ 我的反思
- 我做错了什么
- 我下一步要补什么
- 我对产品方向的长期理解

5️⃣ 求职 CTA
- 想找什么样的岗位
- 联系方式 / 简历链接
```

> 这种文章不仅可以投放求职用，**面试时打开手机给面试官看也是绝佳武器**。

---

> 一句话总结：
> 求职不只是"投简历 → 等回复"，是"在所有触点上反复证明你是值得被记住的候选人"。
> 上面的模板用熟之后，你会发现——
> **0 实习不是你的劣势，是你做差异化叙事的起点。**
