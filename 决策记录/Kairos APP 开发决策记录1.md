## 四、核心逻辑与状态机决策（最终统一版）

本次重点修正、统一了全产品流转逻辑，彻底解决前期逻辑混乱问题，最终定版如下：

### 4\.1 今日待办状态流转（定版）

待办（初始） → 完成 / 逾期 / 删除

- 完成/逾期任务**不可直接删除**，必须恢复待办才可删除

- 删除逻辑：左滑移除 \+ 3s Snackbar 撤销，超时物理删除

- 单日硬限制：**今日待办最大12条**，超出弹窗拦截

### 4\.2 今日记录状态流转（最终对齐Figma）

草稿 → 已保存 → 编辑 / 追加后续 / 引用每日一句 / 删除

- 支持图文记录（最多4张图）

- 支持列表/卡片视图切换、三类分类筛选

- 支持笔记内容引用为全局每日一句

- 删除逻辑与待办统一：3秒撤销、无回收站

### 4\.3 全局数据流转规则（不可修改）

- 每日 **01:30** 自动刷新并锁定昨日数据（可自定义时间）

- 当日首次冷启动强制触发**晨间复盘**，不允许跳过

- 复盘两个唯一操作：带入今日 / 标记逾期

- 空白关闭默认：全部标记逾期

- 复盘完成后解锁今日全部功能

- 待办/记录/每日一句变更，**实时同步所有桌面组件**

## 五、页面与交互最终决策（对齐最新Figma原型）

### 5\.1 底部导航固定5 Tab（定版）

今日 → 今日记录 → 视图 → 小组件 → 我的

### 5\.2 关键交互统一规范（全局一致）

- 所有列表删除统一：左滑删除 \+ Snackbar 3秒撤销

- 任务支持四时间段自由拖拽排序

- 每日一句**唯一入口**：今日页面顶部点击（彻底移除其他入口）

- 小组件仅展示内容，不提供来源配置

### 5\.3 小组件决策

- 支持4种尺寸：1×1 / 2×2 / 3×1 / 3×3

- 可配置：布局、展示条数、刷新时间

- 不可配置：每日一句来源
