\# Kairos App 开发决策记录



本文档记录Kairos App从设计讨论到最终定稿过程中的所有关键决策，包括功能取舍、交互方式、视觉风格、数据模型等，供开发与后续迭代参考。



\---



\## 一、全局架构与风格



| 决策 | 结论 | 理由 |

|------|------|------|

| 底部导航 | 五个Tab：`Today` `Essay` `View` `Widget` `Mine` | 覆盖任务、笔记、视察、小组件、个人中心，功能闭环 |

| 运行模式 | 纯本地存储，无云端、无同步、无账号、无网络 | 优先隐私与离线可用，降低开发复杂度；V2再考虑云同步 |

| 视觉风格 | “素净融入”风格，无独立卡片背景，无阴影，分隔仅用细线和留白 | 保持界面安静、克制，符合“在规律与灵感之间”的理念 |

| 卡片阴影例外 | 仅Mine页的\*\*心情卡片\*\*和\*\*个人卡片\*\*拥有极轻微阴影（elevation 1dp） | 让可拖拽交互区域和身份区域微微浮起，但不破坏整体素净感 |

| 色彩体系 | 去灰色化，用暖棕系替代传统灰色；课题色、紧急度色加深 | 避免冰冷感，增强视觉温度；最终色彩表详见下文 |

\## 二、Essay（笔记）模块



| 决策 | 结论 | 理由 |

|------|------|------|

| 一级分类数量 | \*\*6个\*\*：自我认知、人际互动、亲密与家庭、健康与能量、意义探索、随手记 | 5个深度课题 + 1个自由杂记，“随手记”覆盖碎片化记录，降低门槛 |

| 创建入口 | 点击FAB直接进入编辑页，\*\*默认选中“随手记”\*\*，顶部标签栏可切换课题 | 无弹窗阻断，灵感快速捕获；深度记录可手动切换到课题 |

| 课题强制选择 | 不再强制弹窗选择课题，默认随手记；若未切换即保存，记入Inbox | 零摩擦速记，但保留结构化入口 |

| 行为摘要 | 随手记：\*\*选填\*\*，占位“可选：用一句话概括…”；深度课题：\*\*必填\*\*，占位为对应引导句 | 灵活与规范并存 |

| 二级分类 | 每个深度课题下设3个行为化分类，以\*\*流式chip\*\*展示，单选 | 避免下拉菜单的打断感，直观选中 |

| 关联课题 | 可选0\~4个其他深度课题，以chip一行多选展示 | 解决跨课题事件归属，同时不破坏主课题互斥性 |

| 判断引导句 | 在\*\*快速归类弹窗\*\*中点击课题后展示，起二次确认作用 | 不在创建主流程中打断，仅归类时辅助决策 |

| 笔记状态与Inbox | 随手记保存后进入\*\*Inbox\*\*（15天缓冲），到期自动转\*\*永久杂记\*\*；深度课题直接\*\*发布\*\* | 给灵感一个整理窗口，但不强制；过期自动清理待整理心理负担 |

| 加入项目的影响 | Inbox笔记一旦加入主题项目，\*\*强制立即手动分类\*\*，中断15天自动转正 | 赋予项目意义的笔记需要明确类别 |

| 笔记查看 | \*\*原地展开\*\*，显示完整正文、图片、标签，底部操作条可改课题/改项目/删除，正文\*\*不可编辑\*\* | 保持浏览的沉浸感，防止误改；编辑需进入编辑页 |

| 编辑页 | 编辑页可修改课题、补填摘要、编辑正文、添加图片/标签/心情/项目 | 完整编辑能力，补救归类 |

| 主页面视图切换 | `时间轴` `课题` `项目` 三个视图，通过\*\*侧边栏（选项菜单）\*\* 切换 | 减少顶部常驻占用，界面更纯粹 |

| Inbox入口 | TopBar右侧📥图标，显示数量角标，点击进入Inbox列表 | 收纳所有待归类笔记，不与时间轴混淆 |

| 笔记卡片样式 | 左侧课题色标 + 主课题·二级分类 + 行为摘要 + 标签/项目/时间，\*\*透明背景\*\* | 素净风格，信息层级清晰 |

| 图片板块 | 正文下方可添加图片缩略图，支持多选，带删除 | 丰富记录形式 |

| 标签展示 | 正文上方自适应流式布局，无标签时不占位 | 保持简洁 |

\## 三、Todo（待办）模块



| 决策 | 结论 | 理由 |

|------|------|------|

| 时间块 | 四级：`ANYTIME` `MORNING` `AFTERNOON` `EVENING` | 覆盖全天节奏，兼顾灵活与规划 |

| 紧急度 | 四级：紧急(0)、重要(1)、普通(2)、低(3)，用\*\*空心圆\*\*边框色区分 | 视觉轻量识别 |

| 完成态 | \*\*实心圆 + 白色对勾\*\*，背景色为课题色（无课题默认蓝灰） | 与紧急度脱钩，避免灰色图标 |

| 创建入口 | 各时间块内嵌“+”按钮，直接创建该时段任务；TopBar“+”进入完整创建页（可跨天/重复） | 快捷创建与计划创建分流 |

| 任务排序 | 紧急度升序 > 未完成优先 > 创建时间降序 | 重要紧急先行 |

| 单日上限 | 12条，达上限弹窗提示 | 防止任务过载 |

| 删除 | 左滑删除 + Snackbar 3秒撤销，无回收站 | 符合安卓规范，简化逻辑 |

\## 四、View（视察）模块



| 决策 | 结论 | 理由 |

|------|------|------|

| V1范围 | 仅做数据呈现，\*\*不做模式洞察规则引擎\*\* | 降低初版复杂度，核心是“看” |

| 视图结构 | \*\*今日、本周、本月\*\* 三个时间维度；自定义范围移到后续版本 | 覆盖微观到宏观 |

| 今日视图 | 垂直分区：上半Todo（最多5条任务，按紧急度排序），下半课题笔记（最新3条笔记卡片，点击原地展开） | 一日快照，行动与反思并存 |

| 本周视图 | 左右分页（像摊开的作业本）：左页Todo，右页Essay，七天同步滚动；每日区域有最大高度并可内部滚动；支持\*\*周切换器\*\*（左右箭头） | 对照一周的行动与思考，直观发现空白 |

| 本月视图 | 概览数字 + 课题能量分布（横向柱状条） | 全局统计，课题维度积累可视化 |

| 补录任务 | 本周视图左页每天底部“+ 添加任务”，唤起TaskDetailBottomSheet，预填当天、时间块ANYTIME、紧急度低 | 回顾时顺手补录，不打断浏览 |

| 视觉 | 全区域无卡片背景，分割用淡棕细线，任务行、笔记卡片延用相应模块样式 | 延续素净融入 |

| 滚动行为 | 今日整页滚动；本周左右页整体同步滚动，日区内部滚动；本月整页滚动 | 保证信息完整且对齐 |

\## 五、Mine（个人中心）模块



| 决策 | 结论 | 理由 |

|------|------|------|

| 个人卡片 | 顶部显示Emoji头像 + 昵称 + 记录天数，微阴影，点击编辑 | 本地身份象征，情感归属 |

| 心情卡片 | \*\*拖拽式\*\*Emoji选择，可拖到本周条直接创建当日心情；点击进入历史心情日历；\*\*微阴影\*\* | 惟一的阴影卡片之一，灵动易用 |

| 本周洞察 | 七天打卡圆环 + 连续/总计天数；可通过设置开关 | 轻度数据反馈 |

| 数据总览 | 全部累计：已完成/未完成任务数、今日任务数 | 全局数据一目了然 |

| 备份状态 | 显示上次备份时间，点击跳转数据导出 | 提醒用户定期备份 |

| 年度回顾 | V2实现，V1仅占位一行文字 | 梦想功能，先留入口 |

| 设置入口 | TopBar右侧齿轮⚙️，进入独立全屏设置页（无底部Tab） | 全局设置集中管理 |

| 快捷入口 | 最早设计有Todo/Essay/View/Widget四个快速跳转，\*\*最终移除\*\*，因底部Tab已覆盖 | 避免冗余 |

\## 六、Widget（小组件）模块



| 决策 | 结论 | 理由 |

|------|------|------|

| 尺寸 | 支持4种：1×1、2×2、3×1、3×3 | 适配安卓主流小组件尺寸 |

| 多布局 | 每种尺寸提供1\~3种布局，如计数类、列表类、统计打卡圆环、\*\*一周任务条\*\*、月视图等 | 用户可根据桌面空间和偏好选择 |

| 背景自定义 | 支持\*\*纯色、本地图片、渐变\*\*，可在Widget设置页编辑，实时预览 | 个性化桌面，但保持内容可读 |

| 刷新逻辑 | 默认\*\*每次打开App更新一次\*\*，可自定义固定时间 | 平衡实时性与电量 |

| 跳转 | 各区域（日期、任务、每日一句、圆环）独立跳转到App对应页面 | 精确导航 |

| 交互 | Widget为\*\*只读\*\*，不能直接勾选完成任务 | 系统限制，点击跳转App操作 |

\## 七、设置页面



| 决策 | 结论 | 理由 |

|------|------|------|

| 结构 | 一级列表页，包含通知、主题、Widget设置、数据导出/导入、语言、隐私、缓存、反馈、退出、分享等 | 所有设置集中，条目清晰 |

| 数据导入 | 完整实现，支持JSON/ZIP，含合并/覆盖模式，预览与校验 | 与导出形成闭环，支持试用数据迁移 |

| 主题切换 | 跟随系统/浅色/深色，即时生效 | 满足不同偏好 |

| Widget设置 | 内含尺寸选择栏、布局选择、显示内容开关、刷新时间、背景样式入口 | 小组件所有可定制项集中于此 |

| 心情与反思设置 | 管理心情提醒、心情卡片显示、随机旧笔记浮现等开关 | 控制Mine页及个人化功能 |

\## 八、数据模型与存储



| 决策 | 结论 | 理由 |

|------|------|------|

| 笔记状态机 | `inbox`（随手记暂存）→ `published`（深度课题或永久杂记）；`is\_deleted`软删除 | 清晰覆盖临时与正式、删除与恢复 |

| 任务状态 | `pending` / `completed`，无逾期独立字段（通过日期比较判断） | 简化模型 |

| 心情记录 | 独立`MoodRecord`模型，DataStore存储，不混入笔记 | 轻量独立 |

| 存储方案 | V1 使用 \*\*DataStore Preferences\*\* 序列化 JSON；V2 考虑 \*\*Room\*\* 迁移 | 快速启动，后期升级路径明确 |

| 导入/导出格式 | JSON 统一格式，包含版本号与导出日期 | 数据可恢复、可迁移 |

\## 十、已废弃或延后的想法



| 想法 | 决策 | 原因 |

|------|------|------|

| 长按FAB快速记录 | 废弃 | 改为默认随手记，无需两种入口 |

| 归档机制 | 废弃 | 只保留Inbox，15天自动转存，简单清晰 |

| 图表可视化 | V2 | 先文字统计 |

| 自定义诗词库 | V2 | |

| 任务标签/重复提醒 | V2 | 克制初版功能 |
